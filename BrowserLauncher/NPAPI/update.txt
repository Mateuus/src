Правила создания обновления

Прописываем новую версию в nprt.rc
FILEVERSION 1,0,0,1
PRODUCTVERSION 1,0,0,1
VALUE "FileVersion", "1, 0, 0, 1"
VALUE "ProductVersion", "1, 0, 0, 1"

Создаём копию файла npwiu.xpi под другим уникальным именем, например, с суффиксом версии 
и записываем туда новую dll-ку а так же обновляем файл install.rdf, в котором указываем новую версию:
<em:version>1.0.0.1</em:version>
Выкладываем этот файл рядом с оригинальным npwiu.xpi

В файле update.rdf копируем блок <RDF:li> и в нём:
<em:version>1.0.0.1</em:version> - указываем новую версию плагина
<em:updateLink>http://dl.dropbox.com/u/4068056/wi/npwiu.xpi</em:updateLink> - указываем URL нового xpi-файла

Внимание. файлы должны быть доступны по https-протоколу, иначе firefox не установит обновление

<em:updateInfoURL>https://dl.dropbox.com/u/4068056/wi/update2.htm</em:updateInfoURL> - тут можно сделать страницу с информацией об обновлении
